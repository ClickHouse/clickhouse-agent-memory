The wedge

Vector databases are optimized for unfiltered ANN at high recall. Agent memory retrieval is not that workload.

Agents filter before they rank. They look up past turns for this user, in the last N days, matching this topic vector, ordered by recency, bounded by session. That shape is filter-heavy, time-series-shaped, with a vector-ranking step at the end.

Columnar storage with HNSW handles that shape naturally. Structured indexes cut the candidate set first. Vector similarity ranks the pre-filtered slice. Graph joins expand relationships off the same tables. That is a ClickHouse workload.

The wedge is workload shape, not feature count.
